# DeskEye 启动说明

本文档面向：**在一台新的树莓派上拿到本项目文件夹后，从零完成安装并运行**。  
适用于 Raspberry Pi 5 + CSI 摄像头（OV5647 等 libcamera 设备），系统为 Raspberry Pi OS / Debian Trixie 一类环境。

---

## 一、硬件与系统要求

| 项目 | 要求 |
|------|------|
| 主板 | Raspberry Pi 5（aarch64，建议 4GB+ 内存） |
| 摄像头 | CSI 接口，支持 libcamera（如 OV5647） |
| 系统 | 已刷机、能联网（仅安装阶段需下载模型与 pip 包） |
| 用户 | 能 `sudo`；建议在 `video` 组：`sudo usermod -aG video $USER` 后重新登录 |

---

## 二、获取项目

### 方式 A：从 GitHub 克隆（推荐）

```bash
cd ~
git clone https://github.com/<你的用户名>/deskeye.git
cd deskeye
```

### 方式 B：拷贝文件夹

将整个 `deskeye` 目录复制到树莓派，例如 `~/deskeye`。

> **注意**：不要复制 `.venv/`、`deskeye.env`、`data/deskeye.db` 等本地运行产物；仓库中已用 `.gitignore` 排除。

---

## 三、一键安装

```bash
cd ~/deskeye
chmod +x install.sh    # 首次需要执行权限；也可：bash install.sh
./install.sh
```

脚本会自动完成：

1. 安装系统依赖（GStreamer、libcamera、OpenCV 等）
2. 创建 `.venv` 虚拟环境并安装 `onnxruntime`（规避 PEP 668）
3. 下载人脸与 YOLO 模型到 `models/`
4. 从 `deskeye.env.example` 生成 `deskeye.env`
5. 安装并启用 systemd 服务 `deskeye`

若 `apt` 出现 **raspbian bookworm 源 SHA1 警告**，一般可忽略；若影响安装，可暂时注释 `/etc/apt/sources.list.d/` 中失效的 raspbian 行后重试。

---

## 四、配置摄像头

安装完成后编辑环境文件：

```bash
nano ~/deskeye/deskeye.env
```

将 `DESKEYE_CAMERA_NAME` 改为本机真实路径。查询方式：

```bash
rpicam-hello --list-cameras
# 或
bash scripts/pi_list_cameras.sh
```

复制括号内完整路径，例如：

```
DESKEYE_CAMERA_NAME=/base/axi/pcie@1000120000/rp1/i2c@80000/ov5647@36
```

保存：`Ctrl+O` 回车，`Ctrl+X` 退出。

---

## 五、启动服务

```bash
sudo systemctl start deskeye
sudo systemctl status deskeye
```

看到 **`Active: active (running)`** 即表示运行正常。

### 查看日志

```bash
# 最近 30 条（看完回到命令行）
journalctl -u deskeye -n 30 --no-pager

# 实时跟踪（会持续刷屏，按 Ctrl+C 退出）
journalctl -u deskeye -f
```

`systemctl status` 若进入分页，按 **`q`** 退出。

## 六、使用 Web 仪表盘

在同一局域网内的浏览器打开：

```
http://<树莓派IP>:5000/
```

查看今日专注时长、分心次数、手机检测次数、分数趋势图。页面每 30 秒自动刷新。

查看 Pi 的 IP：

```bash
hostname -I
```

## 七、开机自启

`install.sh` 已执行 `systemctl enable deskeye`。验证：

```bash
sudo reboot
# 重启登录后
systemctl status deskeye
```

## 十、常见问题

**Q：摄像头打不开**  
A：检查 `deskeye.env` 路径、`video` 组权限、`gstreamer1.0-libcamera` 是否已装。

**Q：Web 能开但分数不变化**  
A：确认检测线程在跑：`journalctl -u deskeye -n 20 --no-pager` 中应有 `score=...` 输出。
